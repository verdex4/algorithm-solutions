## Задача

Given an array `nums` of size `n`, return _the majority element_.

The majority element is the element that appears more than `⌊n / 2⌋` times. You may assume that the majority element always exists in the array.

**Example 1:**

**Input:** nums = [3,2,3]
**Output:** 3

**Example 2:**

**Input:** nums = [2,2,1,1,1,2,2]
**Output:** 2

**Constraints:**

- `n == nums.length`
- `1 <= n <= 5 * 104`
- `-109 <= nums[i] <= 109`
- The input is generated such that a majority element will exist in the array.

## Объяснение

Я довольно быстро написал решение со сложностью O(n) по времени и памяти. Идея: собираем словарь частот, находим в нём тот элемент, у которого частота больше чем `n // 2` (такой элемент только один, ведь если бы было два, то их суммарная частота была бы больше `n`, что невозможно).

Решение за O(1) по памяти: алгоритм Бойера-Мура для поиска преобладающего элемента.
Когда я видел follow up в задаче, я понял, что её можно решить за O(1) по памяти. Я понимал, что словари, кортежи, списки, строки использовать нельзя. Скорее всего это число. Но я никак не мог представить, как одним числом хранить состояние всех чисел и их частот.
Идея в следующем: мы уничтожаем пары всех возможных различных чисел. Тогда в конце останется 1 или 0 элементов, притом оставшийся будет преобладающим, потому что его количество в массиве больше n/2, а других меньше n/2. Реализуется это в алгоритме при помощи подсчета оставшихся копий преобладающего элемента:
- встретили 0 -> обновляем кандидата и даем ему 1 копию
- встретили то же число -> увеличиваем количество копий на 1
- встретили другое число -> уменьшаем количество на 1

Если в задаче нет преобладающего элемента, то нужно проверять результат алгоритма через подсчет частоты оставшегося элемента.
При наличии преобладающего элемента алгоритм **всегда** возвращает его, а при отсутствии - любой другой элемент (зависит от списка).

## Сложность

- Время: O(n)
- Память: O(1)
