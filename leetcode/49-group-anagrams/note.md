## Задача

Given an array of strings `strs`, group the anagrams together. You can return the answer in **any order**.
[!] An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, using all the original letters exactly once.

## Решение

### Общие идеи

Это задача на [[Эквивалентность|эквивалентность]] и [[Класс эквивалентности|классы эквивалентности]], поэтому две вещи, которые нам нужны, это:
1. Сами классы
2. Представители классов

Что в данной задаче означает эквивалентность или как понять, что перед нами - анаграмма?
- сортировка анаграмм одинакова: `sorted(s1) = sorted(s2)`
- количества всех букв одинаковы

Как это реализовать?
1. сортировка строк - это O(nlog(n)), где n - длина строки. Не очень
2. здесь можно использовать [[Counter]], но из-за накладных расходов лучше [[Direct address table|таблицы прямой адресации]] (т.е. списки частот букв). Они будут одинаковыми

Что делать с представителями? Как их быстро находить?
1. Если просто пробегаться по списку представителей, это O(m), где m - количество входных строк. Медленно. Нужны хэш-структуры
2. Используем [[Hash map]]. Помним, что ключи должны быть неизменяемыми типами данных, значит нужны либо числа, либо строки, либо кортежи. [[Tuple]] подходят для [[Direct address table]] - выбираем его

Итак, получаем
1. Классы храним как список списков (так просят в ответе)
2. Представителей храним в виде [[Hash map]], где ключ - это [[Tuple|кортеж]], а значение - [[List|список]] представителей
3. Для вычисления ключа заполняем [[Direct address table]]
