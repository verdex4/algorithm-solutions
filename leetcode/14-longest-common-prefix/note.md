## Задача

Write a function to find the longest common prefix string amongst an array of strings.

If there is no common prefix, return an empty string `""`.

**Example 1:**

**Input:** strs = ["flower","flow","flight"]
**Output:** "fl"

**Example 2:**

**Input:** strs = ["dog","racecar","car"]
**Output:** ""
**Explanation:** There is no common prefix among the input strings.

**Constraints:**

- `1 <= strs.length <= 200`
- `0 <= strs[i].length <= 200`
- `strs[i]` consists of only lowercase English letters if it is non-empty.

## Объяснение

Сначала я создал версию с zip. Я сразу думал соединять строки по равным индексам символов, т.е. вертикально. Но в этом методе проблема: приходилось распаковывать список `strs`, получать лишний кортеж. Это быстрое, но не самое эффективное решение.
Лучший вертикальный поиск - через ручную реализацию:
```python
for j in range(max_prefix_len):
    ...
    for i in range(len(strs)):
        ...
```

Все решения в этой задаче O(N * L), где N - кол-во строк, L - макс. длина строки.

Есть и другой способ: горизонтальный поиск.

Я сразу же интуитивно отбросил эту идею, потому что у меня всплыла картина о плохой строке, которая может попасться и алгоритм сломается. Так бывает часто, но не в этот раз. Здесь та самая плохая строка - это самая важная строка. `ttat, ttl, a, tt`. Здесь не ломается логика.
Идея: мы динамически вычисляем prefix. Для начала берем всю первую строку в качестве префикса, а потом идем по всем символам этой строки и сравниваем с символом следующей. Если j >= длины, то обрезаем. И если несовпадение, то тоже обрезаем. Мне сложно дался этот способ.

## Сложность

- Время: O(N * L), N - кол-во строк, L - макс. длина 
- Память: O(1)
