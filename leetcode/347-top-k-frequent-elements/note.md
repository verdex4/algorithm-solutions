## Задача

Given an integer array `nums` and an integer `k`, return _the_ `k` _most frequent elements_. You may return the answer in **any order**.

## Решение

Считаем словарь частот. А потом думаем: "как можно вернуть k ключей с максимальными значениями?". 
Вопрос сложный, используем подсказку: мы можем пробежаться по словарю за O(n) (нас устраивает). Что можно сделать, если пробежаться по словарю? Нужна сложность O(1) в цикле.

Ответ - **сгруппировать элементы по значениям**. Зачем? Тогда мы за O(1) создадим упорядоченную структуру данных. Нам останется пройти k элементов с конца. 
Что это за структура? Словарь не подойдет, т.к. он будет неупорядочен. Нужен список. Это [[Direct address table]]. Индекс - частота. Тогда последний индекс - максимальная частота.

Общая идея: словарь частот [[freq]] + группировка по значениям в список списков ([[Direct address table]])
